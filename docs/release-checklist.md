# Release Checklist

- [ ] Feature has clear acceptance criteria.
- [ ] Threat model updated when data flow or trust boundary changes.
- [ ] Tests pass locally.
- [ ] Security header check passes locally.
- [ ] CI passes on pull request.
- [ ] Dependency audit has no unresolved high or critical findings.
- [ ] Container builds and health check passes.
- [ ] Rollback plan is documented.
- [ ] Monitoring owner confirms post-release watch.
- [ ] Release evidence is linked in the pull request.
